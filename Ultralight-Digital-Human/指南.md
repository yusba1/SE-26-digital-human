# 数字人模型训练指南

## 一、训练流程

**准备训练素材**
1. **训练视频**：一段人物正面说话的视频（前 20-30 秒不说话，总时长 3min 左右，人物不能有大范围移动，画面不能跳帧，无杂音、光线均匀）。视频可以真人录制，也可以使用 AI 生成。
2. **测试音频**：一段用来测试效果的音频（mp3/wav/m4a 均可），建议语种与训练视频一致。

**把训练素材放到指定路径下**
```bash
mkdir -p data_utils/张三
cp 你的训练视频.mp4 data_utils/张三/video.mp4
cp 你的测试音频.mp3 data_utils/张三/test_audio.mp3
```

**一键运行 `run_pipeline.sh`**
```bash
bash run_pipeline.sh
```

脚本会自动完成预处理、训练、推理、打包、ONNX 转换。  
最终重点产物：

- `checkpoints/<项目名>/final.pth`（主模型）
- `checkpoints/<项目名>/<项目名>_<asr>.onnx`（ONNX 模型）
- `data_utils/<项目名>/<项目名>_test.mp4`（有声推理视频）
- `data_utils/<项目名>/<项目名>_<start>_<end>.zip`（图片+关键点打包）

## 二、训练细节和参数调整

编辑 `run_pipeline.sh` 顶部参数（只改你需要的项）：

```bash
# ================== 项目配置（必填） ==================
PROJECT_NAME="Veo1"
VIDEO_FILE="veo1_20fps.mp4"
TEST_AUDIO_FILE="veo1_test.wav"

# ================== 训练超参数 ==================
ASR_MODE="wenet"
EPOCHS=80
BATCH_SIZE=32
LEARNING_RATE=0.001
...
```

调好后执行：
```bash
bash run_pipeline.sh
```

脚本实际流程如下（与当前 `run_pipeline.sh` 对齐）：

| 步骤 | 说明 | 产出 |
|------|------|------|
| Step 1 | 视频预处理（检查并补齐：转帧率、提取音频/图片/关键点/音频特征） | `full_body_img/`、`landmarks/`、`aud_wenet.npy` 或 `aud_hu.npy` |
| Step 2 | SyncNet 训练（`USE_SYNCNET=True` 时执行，否则跳过） | `checkpoints/<项目名>_syncnet/final.pth` |
| Step 3 | 主模型训练 | `checkpoints/<项目名>/final.pth` |
| Step 4 | 推理合成有声视频（含测试音频转 16k WAV + 特征提取） | `data_utils/<项目名>/<项目名>_test.mp4` |
| Step 5 | 按推理帧范围打包图片与关键点（若未设结束帧则按 `PACK_COUNT`） | `data_utils/<项目名>/<项目名>_<start>_<end>.zip` |
| Step 6 | 模型转换为 ONNX | `checkpoints/<项目名>/<项目名>_<asr>.onnx` |
| Step 7 | 输出总结 | — |

## 三、参数详情（当前脚本默认值）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `ASR_MODE` | `wenet` | 音频特征类型：`wenet` 要求视频 20fps，`hubert` 要求 25fps |
| `EPOCHS` | `80` | 主模型训练轮次 |
| `BATCH_SIZE` | `32` | 主模型批次大小 |
| `LEARNING_RATE` | `0.001` | 主模型初始学习率 |
| `LR_DECAY_STEP` | `20` | 每 N 轮学习率衰减一次 |
| `LR_DECAY_FACTOR` | `0.5` | 学习率衰减倍率 |
| `SAVE_INTERVAL` | `30` | 主模型每 N 轮保存一次 |
| `NUM_WORKERS` | `4` | DataLoader 工作进程数 |
| `USE_SYNCNET` | `True` | 是否启用 SyncNet |
| `PERCEPTUAL_LOSS_WEIGHT` | `0.01` | 感知损失权重 |
| `SYNC_LOSS_WEIGHT` | `10` | SyncNet 损失权重（启用时生效） |
| `SEE_RESULTS` | `False` | 是否可视化训练中间结果 |
| `SYNCNET_EPOCHS` | `15` | SyncNet 训练轮次 |
| `SYNCNET_BATCH_SIZE` | `32` | SyncNet 批次大小 |
| `SYNCNET_LEARNING_RATE` | `0.001` | SyncNet 学习率 |
| `SYNCNET_SAVE_INTERVAL` | `20` | SyncNet 保存间隔 |
| `INFERENCE_START_FRAME` | `1500` | 推理起始帧 |
| `INFERENCE_END_FRAME` | `1650` | 推理结束帧（`-1` 表示到最后一帧） |
| `PACK_COUNT` | `150` | 未设置结束帧时，从起始帧打包数量 |

> 注：这个推理起始帧和结束帧根据你准备的视频素材，找一段人物比较稳定，没有卡顿适合推理的时间就行，大概 20s 左右，手动计算出帧数后填上即可
